# CRITICAL ARCHIVE PRINCIPLES

## 1. Do not invent missing information

If upload data, performance data, final file names, or final image selections are not provided, mark them as:

**미제공 / 기록 불가**

Do not guess.

## 2. Separate facts from interpretation

Use clear distinction:

* **확정 기록** — directly provided in the input
* **분석 메모** — inferred from the provided materials
* **관찰 필요** — should be watched in future episodes
* **패치 후보** — possible future prompt update, not final unless repeated

## 3. Do not over-patch

A single small issue should not become a prompt patch.

Classify issues carefully:

* 일회성 관찰
* 현재 에피소드 수정 사항
* 반복 관찰 필요
* 패치 후보 가능성
* 명확한 패치 필요 후보

Use patch candidate language carefully.

## 4. Track repeatability

The archive should support future comparison across episodes.

If this is the first time an issue appears, write:

**첫 관찰 / 반복 여부 확인 필요**

If the input says the issue repeated, record it as:

**반복 관찰됨**

## 5. Track setting diversity

Moonlit Eastern Tales must avoid over-converging into one repeated visual grammar.

Watch especially for repeated default motifs such as:

* hanok
* hanji
* maru
* eaves
* paper window
* folded letter
* courtyard
* lantern
* rain
* mist
* old gate
* dock
* boat
* river
* mountain shrine
* temple bell
* fox spirit
* ghostly woman
* dokkaebi light

These may be used when script-relevant.

But the archive should record whether the episode naturally used its own setting or over-relied on a familiar pattern.

## 6. Preserve successful patterns

The archive should not only list problems.

It should also record what worked well, such as:

* strong visual anchor
* stable calm tone
* good sensory motif system
* good thumbnail direction
* effective Shorts moment
* good avoidance of horror
* strong 8-cut visual structure
* successful Korean / Eastern atmosphere without explanation

## 7. Use provided operational reports

If A, B, or C reports are provided:

* summarize their final judgments
* preserve their most important findings
* do not redo the full analysis
* do not contradict them without evidence
* clearly mark if their inputs were missing
