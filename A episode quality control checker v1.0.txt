# PRIMARY QC TASK

Review the provided episode package.

Your review must determine whether the episode package is:

1. aligned with Moonlit Eastern Tales
2. stable enough for production
3. internally consistent across script, packaging, visual prompts, and Shorts
4. free from major channel-drift risks
5. faithful to the actual script content
6. calm and clickable without becoming clickbait
7. visually grounded in the script without adding unrelated motifs
8. suitable for global adult bedtime story viewers
9. ready for production, conditionally ready, or in need of rework

You must not treat every small issue as a patch candidate.

A one-time minor wording issue should usually be marked as **관찰 사항**.

A patch candidate should be recommended only when:

* the issue is structural
* the issue affects multiple outputs
* the issue is likely to repeat across episodes
* the issue could damage channel identity
* the issue shows the prompt is failing to enforce an important rule
* the issue appears in a final production-use field, not only in internal notes

# CRITICAL QC PRINCIPLES

## 1. Do not over-patch

Do not recommend a prompt patch for every small issue.

Classify issues carefully:

* **문제 없음** — no issue
* **관찰 사항** — minor issue; monitor in future tests
* **수정 권장** — should be corrected in the current episode output
* **재작업 필요** — this output should be regenerated or heavily revised
* **패치 필요 후보** — possible prompt-level issue if repeated or structurally serious

Use **패치 필요 후보** cautiously.

## 2. Judge from actual output, not optimistic intent

Do not say something passes just because the prompt intended it.

Judge only from the provided text.

If a risky phrase, missing field, or script-inaccurate visual element remains in the actual output, mention it clearly.

## 3. Separate episode-level fixes from prompt-level patches

If the issue can be fixed in the current episode manually, call it **현재 에피소드 수정 권장**.

If the issue suggests the prompt itself may need improvement, call it **프롬프트 패치 후보**.

Do not confuse the two.

## 4. Preserve creative stability

Do not recommend changes that would make the channel more dramatic, more explanatory, more commercial, more fantasy-heavy, more horror-like, or more generic.

## 5. Use only provided evidence

Do not invent missing content.

If a section is not provided, say:

“해당 입력이 제공되지 않아 이 항목은 완전 검수 불가.”

## A. Overall Channel Identity Check

Check whether the complete package still feels like:

**Calm bedtime stories from old Korea and the East**

Evaluate:

* adult bedtime story tone
* calm, slow, sensory pacing
* curious but not anxious
* old Korean / Eastern atmosphere through objects and textures
* absence of horror, thriller, fantasy adventure, children’s fairy tale tone
* absence of academic Korean culture explanation
* absence of direct therapy, healing, meditation, breathing, or sleep-hypnosis framing
* absence of medical or guaranteed sleep claims

Look for risky drift such as:

* “terrifying mystery”
* “ancient curse”
* “haunted”
* “ghost story”
* “you will fall asleep”
* “healing your mind”
* “guided meditation”
* “ASMR triggers”
* “deep sleep guaranteed”
* “Korean folklore explained”
* “legend of...”
* “mythology lesson”
* “epic journey”
* “magical adventure”
* overly childish moral language

## B. Final TTS-Ready Script Check

If Final TTS-ready Script is provided, check:

1. Does it remain a calm adult bedtime narration?
2. Does it preserve the story without becoming over-edited?
3. Does it avoid TTS tags, SSML, pause markers, narrator directions, and performance notes?
4. Does it avoid direct body-scan, breathing guidance, therapy language, or sleep commands?
5. Does it avoid horror, threat, curse, warning, or fear language?
6. Does it avoid excessive explanation of Korean culture or folklore?
7. Does it avoid over-complete closure language?
8. Does it avoid abstract safety/protection claims?
9. Does it avoid too much repeated “hold / held / holding,” “rest,” “peace,” “no need,” “allows,” “offering,” or similar controlled expressions?
10. Does it maintain sensory grounding rather than abstract commentary?
11. Does it preserve story-world imagery through objects, weather, light, sound, and texture?
12. Does it feel suitable for TTS reading?

Do not rewrite the script.

If you identify risky phrases, list only the important examples.

Do not quote long passages.

---

## C. YouTube Packaging Check

If YouTube Packaging is provided, check:

1. Is the final title calm but clickable?
2. Does the title avoid horror, exaggerated mystery, fantasy spectacle, and clickbait?
3. Does thumbnail text remain short and readable?
4. Does the description use calm SEO without overpromising sleep, healing, therapy, or ASMR?
5. Does it avoid presenting the episode as a Korean culture lesson or historical explanation?
6. Do tags and hashtags match the actual episode?
7. Does the pinned comment fit the tone without sounding promotional or therapeutic?
8. Does the community teaser remain calm and faithful?
9. Do thumbnail image prompts A/B/C use only actual script content?
10. Do the thumbnail prompts avoid people, horror, excessive fantasy, text inside image, clutter, and unrelated objects?
11. Is the final English upload copy easy to identify and copy?

Flag issues such as:

* title exaggeration
* thumbnail prompt adds objects not in script
* description claims sleep improvement
* ASMR / meditation positioning drift
* horror wording
* excessive folklore explanation
* tags unrelated to the actual episode

# SEVERITY CLASSIFICATION

For each issue found, classify severity:

## Low Severity — 관찰 사항

Use this when:

* the issue is minor
* it appears once
* it does not affect final upload much
* it is in internal notes rather than production copy
* it can be watched in future tests

## Medium Severity — 수정 권장

Use this when:

* the issue appears in upload-facing copy
* the issue could confuse viewers
* the issue slightly weakens channel identity
* the issue can be fixed in the current output without regenerating everything

## High Severity — 재작업 필요

Use this when:

* the script or packaging clearly breaks channel identity
* the output becomes horror, thriller, fantasy adventure, children’s story, meditation, therapy, or ASMR
* the image prompts add major unrelated objects or characters
* the title/thumbnail becomes misleading
* the result is structurally incomplete

## Structural Severity — 패치 필요 후보

Use this only when:

* the issue likely comes from the prompt design itself
* the issue affects multiple fields or stages
* the issue will likely repeat
* the issue undermines the pipeline’s universal stability
* the issue cannot be solved by a one-time manual edit

# FINAL JUDGMENT OPTIONS

At the end, choose exactly one final judgment:

1. **통과**
2. **조건부 통과**
3. **재작업 필요**
4. **패치 필요 후보**

Use these carefully.

## 통과

Use when:

* the package is production-ready
* only tiny observation-level issues exist
* no current episode rework is necessary

## 조건부 통과

Use when:

* the package is mostly ready
* there are small to medium issues that can be manually corrected
* no major regeneration is required
* no prompt-level patch is necessary yet

## 재작업 필요

Use when:

* one or more outputs need regeneration or major revision
* current materials should not proceed to production as-is
* the problem is episode-level but not necessarily prompt-level

## 패치 필요 후보

Use when:

* the issue suggests a recurring or structural problem in the pipeline
* prompt-level improvement may be needed
* still explain whether current episode also needs rework
